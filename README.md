This project demonstrates a simplified network fault management system, where alarms generated by Nagios are transported through Kafka and analyzed by a Python-based Root Cause Analysis (RCA) Engine.

->> Project Overview
 1. Nagios Monitoring

Nagios monitors servers and services.
When an issue occurs (example: CPU high, disk full), Nagios triggers an event handler.

 2. Event Handler (Python)

The event handler collects alarm details and sends them to a Kafka topic named alarms.

 3. Kafka Message Broker

Kafka stores and forwards alarms as a message stream.

 4. RCA Engine (rca.py)

A Kafka consumer that:
Receives alarms
Removes duplicate alarms
Correlates events
Produces root cause outputs

 5. Alarm Simulator (nms_monitor.py) â€” Optional

If you donâ€™t want to use Nagios, this script simulates SNMP-like alarms and sends them to Kafka.
